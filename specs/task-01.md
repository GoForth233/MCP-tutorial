# Task 01 — 落地页改造规划

## 目标
- 将现有 Supabase + Next.js 模板调整为教程型咖啡豆介绍网站的落地页基础框架。
- 删除首页复杂布局，只保留认证入口、导航与空白提示文案。
- 推进全站中文化与统一背景色。

## 执行步骤
- [x] **梳理现状**
  - 阅读 `app/layout.tsx`, `app/page.tsx`、`components` 目录下的导航/认证相关组件，记录需要保留的逻辑与需要替换的英文文案。
  - 确认项目中是否已经集成 `lucide-react`，若缺失则在 `package.json` 中新增依赖并安装。

- [x] **全局样式调整**
  - 在 `app/globals.css` 中为 `body` 或根节点设置背景色 `#FFFEFC`，保证全站继承该色值。
  - 检查是否存在与背景色冲突的局部样式，必要时删除或覆盖。

- [x] **Navbar 组件封装**
  - 在 `components` 目录新增（或重构）一个可复用的 `Navbar` 组件，布局要求：顶部居中、包含「首页」「咖啡豆」两个链接。
  - 结合 Next.js 路由生成链接，预留后续页面扩展位置；所有导航文案使用中文。

- [x] **首页 `app/page.tsx` 改造**
  - 移除现有内容，仅保留：
    - 顶部引入并渲染新的 `Navbar`。
    - 页面主体为空白背景，中间水平居中显示文案「落地页」。
  - 右上角保留 Supabase 认证入口：未登录展示「登录」「注册」按钮；已登录显示 `lucide-react` 默认用户头像图标。
    - 点击头像打开 Popover，内容包含当前邮箱与「退出登录」按钮，中文化提示与按钮文本。
  - 确保 Popover 与按钮样式与背景色协调，必要时适配 CSS。

- [x] **其它相关页面同步**
  - 移除 `app/protected` 相关页面，仅保留教程所需的公开路由。
  - 确认其它路由展示中文文案「落地页」，避免残余英文内容。

- [x] **中文化改造**
  - 全面搜索项目中的英文 UI 文案（组件、页面、提示语），逐项替换为对应中文。
  - 保留变量名、代码注释原有语种，重点调整用户可见内容。

- [x] **验证**
  - 本地运行开发服务器，验证登录/注册流程、头像 Popover 状态切换、导航布局与背景色。
  - 自查所有页面无残留英文文本，交互符合要求后提交。
